1、用户在前端发起提现申请后，后台生成一条记录；

2、后台生成记录后对 金额数目 进行校验，若出现数值不符合的则标记为风险订单（红色字体），校验如下：

    1、“钱包余额” = （累积已解冻馆长收益 + 累积下线馆长推广收益 + 累积借阅服务收益）-历史已提现金额，

        ① 馆长收益风险：馆长已解冻馆长收益金额 与 实际成交订单获得收益不符合；

        ② 下线馆长推广收益风险： 馆长已解冻推广收益金额 与 下线馆长实际成交订单数不符合；

        ③ 借阅服务收益风险：馆长借阅服务收益金额 与 实际成果借阅订单不符合；

        ④ 历史已提现金额风险：馆长历史累积已提现金额 与 成功发起提现并成功打款金额不符合；

    2、实际打款金额：

        ①若提现金额 ≤ 800元，则 打款金额 = 提现金额；

        ②若提现金额＞800元，则 提现金额超800元部分需收取20%税费，即打款金额 = 提现金额 - （提现金额 -           

           800）*20%。
